Preguntas

  1. ¿Qué es el DOM?
    A: El DOM (Document Object Model) es una representación estructurada en forma de árbol de un documento HTML o XML. Proporciona una interfaz que permite acceder y manipular los elementos de un documento web de manera dinámica. El DOM representa cada elemento HTML como un nodo en el árbol, lo que permite modificar su contenido, atributos y estilos, así como responder a eventos.
  2. ¿Qué es un nodo?
    A: Un nodo en el contexto del DOM es cada uno de los elementos que conforman el árbol de estructura del documento. Puede representar elementos HTML, atributos, texto, comentarios u otros tipos de contenido. Cada nodo tiene relaciones jerárquicas con otros nodos, como nodos padre, nodos hijos y nodos hermanos.
  3. ¿Cómo se accede a un elemento del DOM?
    A: Para acceder a un elemento del DOM, se pueden utilizar diferentes métodos y propiedades según el caso. Algunos ejemplos comunes son:
    • getElementById: Permite acceder a un elemento por su ID único.
    • getElementsByClassName: Permite acceder a elementos por su clase.
    • getElementsByTagName: Permite acceder a elementos por su nombre de etiqueta.
    • querySelector: Permite acceder a un elemento utilizando selectores CSS.
    Estos métodos devuelven una referencia al elemento que se puede almacenar en una variable y luego se puede manipular.
  4. ¿Cómo se modifica un elemento del DOM?
    A: Para modificar un elemento del DOM, se pueden utilizar propiedades y métodos disponibles en el elemento. Algunas formas comunes de modificar un elemento son:
    • Modificar el contenido del elemento utilizando la propiedad innerHTML o textContent.
    • Cambiar atributos del elemento utilizando la propiedad setAttribute.
    • Modificar estilos del elemento utilizando la propiedad style.
    • Agregar o eliminar clases utilizando las propiedades classList o className.
  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?
    A: Las diferencias principales son:
    • getElementById, getElementsByClassName y getElementsByTagName son métodos más antiguos y devuelven una colección HTML o un solo elemento específico en base a su ID, clase o etiqueta.
    • querySelector y querySelectorAll son métodos más modernos que permiten seleccionar elementos utilizando selectores CSS, lo que brinda una mayor flexibilidad y poder de selección.

Conclusiones

Experiencia general

ChatGPT y github copilot son ambas herramientas muy poderosas. Mientras este aprendiendo, espero no usarlas para que pueda aprender bien, pero la verdad es que ya sepa hacer algo, el no usarlas no solamente me parece innecesario, si no puede significar ser una desventaja laboral. Si un men que solo sabe usar ChatGPT puede hacer código mas rápido que yo, no veo porque no hacerlo yo. Es como usar un compilador. El binario existe, pero ya hay nuevas formas de trabajo.

¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

Copilot, como dice su nombre, es mejor como un copiloto. Estas en tu editor y mientras estas programando, te termina la idea. Mientras tanto, ChatGPT es mejor para cuando no sabes que hacer. Puedes pedirle que te de ideas, y puedes pedir que arregle código dándole suficiente contexto.
